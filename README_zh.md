# MICP 课件包生成指引

[**English**](./README.md)

这个仓库不是 Moodle 插件本身。

它是一套给 AI 助手使用的通用指引，用来生成可以上传到 MICP for Moodle 的课件包。

最简单的理解方式是：

- Moodle 插件负责在课程里运行活动
- 这个仓库负责告诉 AI 助手怎么生成那个活动包

## 这个仓库是给谁用的

它适合这些人：

- 想让 AI 帮自己做互动课件的教师
- 负责课程建设的团队
- 需要批量生成课件包的开发者或教研人员

它特别适合帮助生成：

- 分步讲解页
- 多步骤学习活动
- 带可视化内容的互动解释
- 可自动评分的练习活动
- 同时包含自动评分和人工复核内容的活动

## 它会产出什么

目标产物是这样一个课件包：

```text
output-dir/
├── index.html
├── micp-scoring.json
└── assets/
    └── micp.js
```

然后把这个课件包上传到 MICP Moodle 插件里即可使用。

## 不懂 AI 工具也可以怎么理解

如果你不关心 AI 工具的技术细节，可以把这个仓库理解成：

1. 把这套说明交给 AI 助手。
2. 让它按你的教学主题生成一个互动课件。
3. AI 助手输出一个 MICP 课件包。
4. 你再把这个课件包上传到 Moodle。

所以，这个仓库本质上就是一套“教 AI 怎么帮你生成课件包”的说明书。

## 它是不是只能给 Codex 用

不是。

这套方法不是绑定某一个厂商的。它可以迁移到这些工具里：

- Codex
- OpenCode
- OpenClaw
- Claude Code
- 其他能读取仓库说明并生成文件的代理式工具

当前仓库里的 `SKILL.md` 是一种具体封装形式，但方法本身是通用的。

## 它为什么对教学有意义

很多教师并不是没有好活动想法，而是实现过程太费时间。

常见卡点包括：

- HTML 页面不会写
- 不知道怎么记录学生的操作
- 不知道怎么设计评分规则
- 最后做出来的东西不容易真正放进 Moodle 使用

这个仓库的目的不是“让页面更炫”，而是减少这些制作成本，帮助教师更快得到一个真的能上传、能运行、能评分的互动活动包。

## 基本用法

### 方式 1：配合 AI 助手使用

把这个仓库或 `SKILL.md` 交给 AI 助手，然后给出你的需求，例如：

```text
为初中生制作一个关于光合作用的 MICP 互动课件。
```

AI 助手应当生成：

- `index.html`
- `micp-scoring.json`
- 所需资源文件

### 方式 2：手工复用模板

如果你有开发者支持，也可以直接复用仓库里的模板、模式和资源来手工制作。

## 仓库结构

```text
.
├── SKILL.md
└── references/
    ├── assets/
    ├── patterns/
    ├── templates/
    └── quick-reference.md
```

- `SKILL.md` 是主要指令文件
- `references/` 存放模板、互动模式和资源文件
- 本 README 是给人看的仓库说明

## 它和 Moodle 插件是什么关系

这个仓库负责生成课件包。

Moodle 插件仓库负责在 Moodle 里运行课件包、记录学习者行为、评分并生成报告。

两个仓库彼此相关，但职责不同。
