# MICP Authoring Skill

[**English**](./README.md)

> **传统方式：** 教师有一个互动教学想法，但很快卡在 HTML、打包结构、交互记录和评分规则上。
> **MICP 方式：** 让代理直接生成一个可以上传到 MICP 的完整课件包。

`MICP Authoring Skill` 是一个面向 `mod_micp` 的通用 AI skill / 指令包，用来生成完整可部署的 Moodle 互动课件包。

它不只适用于 Codex，也可以迁移到 OpenCode、OpenClaw、Claude Code 以及其他能读取仓库说明、生成文件并复用参考模板的代理式工具。当前仓库里的 `SKILL.md` 是一种具体封装形式，但背后的课件生产方法本身是通用的。

这个 skill 只做一件事：生成一个可以直接交给 `mod_micp` 使用的课件包：

```text
output-dir/
├── index.html
├── micp-scoring.json
└── assets/
    └── micp.js
```

## 为什么教师和课程团队会在意

教师通常很清楚自己想让学生完成什么活动，真正麻烦的是把这个想法落成一个能进课程、能评分、能复用的活动包：

- 页面交互要自己写
- 打包结构容易出错
- 评分规则和前端行为对不齐
- 做出来的页面看起来像演示稿，不像可运营的课程活动

这个 skill 的价值就在于缩短这段距离。它帮助 AI 代理生成的不只是“看起来能点的 HTML”，而是更接近教学可用状态的活动包：

- 结构上能上传到 `mod_micp`
- 交互上能留下可评分的学习证据
- 规则上能支撑服务端评分
- 表达上能支持讲解、探索、练习、反思等不同教学任务

对教师来说，更实际的结果是：从“我有一个教学活动想法”到“我有一个可以上传进 Moodle 的活动包”，中间需要的人力和前端制作成本会明显下降。

## 核心工作流

```text
教学活动想法 -> 代理生成 MICP 课件包 -> 上传到 MICP -> 在 Moodle 中交付给学生
```

这个仓库承诺的不是泛化的 HTML 生成，而是能进入真实教学流程的课件包生成。

## 仓库结构

```text
.
├── SKILL.md
└── references/
    ├── assets/
    ├── patterns/
    ├── templates/
    └── quick-reference.md
```

- `SKILL.md` 是当前仓库中的核心运行说明
- 同一套方法也可以迁移到其他 agent 工具的 skill / prompt 体系中
- `references/` 存放模板、交互模式和运行时资源
- 本 README 是仓库层说明，不属于 skill 运行时加载契约

## 适用方式

当你需要让 AI 代理生成一个兼容 Moodle `mod_micp` 的互动课件包时，就可以使用这个仓库。

在不同环境里，这可能意味着：

- 把它安装为 Codex 兼容 skill
- 把 `SKILL.md` 改写成其他代理工具的指令格式
- 直接复用其中的 references、templates 和 assets 作为通用课件生成包

典型触发场景包括：

- 为 mod_micp 创建互动 HTML 活动
- 生成 MICP 课件包
- 创建渐进式披露课程
- 制作可以上传到 Moodle 的互动课件

## 与 mod_micp 的关系

这个仓库负责“课件包怎么生成”。
`mod_micp` 插件仓库负责“课件包怎么在 Moodle 里承载、记录、评分、回写成绩册”。

两者是互补关系，不是同一个仓库里的不同目录。

## 适合的教学产出

这个 skill 特别适合帮助教师或课程团队生成：

- 分步引导式活动
- 带过程记录的互动讲解页
- 先探索后提交证据的学习任务
- 同时包含自动评分和人工复核的混合型活动
- 可持续复用、迭代的课件包

它真正改进的不是“让页面更炫”，而是让更高质量的互动活动更容易进入真实教学流程。
